# Ключевые решения

1. Использован чистый PHP без фреймворков — проект локальный, проще развернуть.
2. Авторизация через сессию, роли: диспетчер, мастер, клиент.
3. Защита от гонки реализована через проверку статуса в SQL UPDATE.
4. Статусы заявки — ENUM (new, assigned, in_progress, done, canceled) для простоты и безопасности.
5. Отдельные таблицы: users (мастера и диспетчер) и requests.
6. Миграции и сиды вынесены в отдельные папки для удобного развертывания.
7. UI минималистичный, единый стиль для всех страниц (header/footer, стили CSS).
